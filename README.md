## Список технологий проекта

  
- ## Шифрование и хэш-суммы
  > На компьютере пользователя данные аккаунта хранятся в зашифрованном виде. Для шифрования используется SHA256. 
  > В некоторых случаях при передаче избражение по сети используется Base64. На стороне севера хранится хэш-сумма пароля аккаунта c использованием функции MD5.
  > Так же, при каждом запуске игры все файлы сверяются по хэшу с копиями на сервере

- ## Размытие по гауссу
  > Дизайн UI требовал размытие заднего фона, из-за чего и было написано несколько алгоритмов размытия по гауссу.
  > Присутствует как обычный алгоритм, так и оптимизированный (Box blur). Позже, в целях чистоты кода, была использована
  > [библиотека с готовым решением](http://www.jhlabs.com/ip/blurring.html)

- ## OpenGL (JOGL)
  > Для отображения скина игрока в профиле создаётся 3D моделька. Она имеет анимации, а так же возможность ношения плаща.
  > Исходный код был адаптирован для HiDPI

- ### Античит и интегация в процесс игры
  > Для гарантии честной игры были произведены следующие действия. При подключении к серверу в игре, автоматически проверяется целостность всех файлов клиента. 
  > Это позволяет избежать модификации клиента после запуска лаунчером, во время загрузки игры. 
  > Помимо этого, в фоне античит сканирует процесс игры на подключившиеся dll библиотеки, что позволяет исключить использование инжекторов.

- ## WinAPI
  > Используется для изменения иконки и заголовка окна игры, а так же для получения Id процесса на Windows.
  > Ещё имеется возможность получить текущую тему системы (тёмная или светлая), но применялась только для тестов

- ### Java Native Access (JNA)
  > Используется для доступа к нативным функциям ОС, таким как WinAPI (user32.dll)

- ### HiDPI в Swing 
  > Обычные UI элементы, встроенные в Swing корректно не поддерживают повышенный масштаб интерфейса. 
  > Это вызывает эффект мыльности на мониторах с высоким разрешением. 
  > Для решения этой проблемы было переписано большое количество элементов UI с собственной отрисовкой

- ### Сетевой API
  > В процессе разработки был создан очень гибкий и оптимизированный сетевой API (возможно, самая лучший код проекта).
  > Для доступа к большинству методов неоходим токен доступа, который в свою очередь можно получить после авторизации.
  >
  > Весь API разделяется на следующие модули:
  >   - Манипуляции с аккаунтом (Auth, Profile, Profiles, Skins)
  >   - Информация (About, Launcher, Github, Instagram, Vk, Youtube)
  >   - Файлы (Clients, Skins)
  >
  >
  > На стороне сервера обеспечено минимальное потребление памяти для каждого HTTP запроса.
  > В то же время на стороне клиента создан програмный интерфейс с обязателным учётом любых ошибок.
  > Это значит, что разработчик обязан предусмотреть все варианты поведения запроса

- ## Парсинг
    * ### YouTube
      > Доступна возможность узнавать количество подписчиков, а так же получать иконку канала.
      > Так же можно получать список последних видео с заголовком и превью в высоком разрешении
    * ### VK
      > Имеется возможность получения описания, иконки, а так же количества участников сообщества.
      > Вместе с этим можно получать последние записи с разными типами - Текст, Фото, Видео, Сниппет
    * ### Instagram
      > Обычное получение информации о последних фотографиях пользователя, используя небольшую лазейку в API инстаграма

- ## Discord IPC
  > Отображение информации об игроках сервера Minecraft в Discord
